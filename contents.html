class gbsdr.Graph(nodal_coords, edges, weights=None)

Bases: object

Initialize the representation of a graph.

Parameters:

  • nodal_coords (array-like of shape (n_nodes, n_dims)) – Coordinates of each node in the structure. Each row corresponds to a node and contains its spatial coordinates.

  • edges (array-like or list of lists) – List of edges in the graph. Each edge is represented as a pair of node indices.

  • weights (array-like) – Edge weights representing axial forces.

Notes

This initializer constructs the full graph representation of a structural system, including geometry, topology, and optionally applied loads and boundary conditions.

property ClosenessCentrality

Closeness centrality of the graph.

Closeness centrality measures how close a node is to all other nodes in the graph based on shortest path distances. Nodes with higher values can reach all other nodes more quickly on average.

Higher values indicate nodes that are more centrally located in terms of graph distance.

Returns:

Mapping from node to closeness centrality value in the range [0, 1].

Return type:

dict

Notes

Computed using NetworkX closeness centrality. See: https://en.wikipedia.org/wiki/Closeness_centrality.

Degree centrality measures the fraction of nodes a given node is directly connected to. It is the simplest form of centrality and reflects immediate connectivity in the network.

Nodes with high degree centrality are directly connected to many other nodes.

property Density

Graph density.

Graph density measures how close the graph is to being complete, defined as the ratio of existing edges to the maximum possible number of edges.

  • Density = 0 → no edges in the graph

  • Density = 1 → fully connected (complete graph)

  • Intermediate values indicate how sparsely or densely connected the graph is

Returns:

Graph density in the range [0, 1].

Return type:

float

Notes

Computed using NetworkX. See: https://networkx.org/documentation/stable/reference/generated/networkx.density.html

property Diameter

Graph diameter.

The diameter of a graph is the longest shortest-path distance between any pair of nodes. It represents the greatest minimal distance required to travel between two nodes in the network.

  • The diameter represents the worst-case shortest-path distance in the graph.

  • It provides a measure of the “size” of the graph in terms of connectivity.

  • Larger values indicate more spread-out or less tightly connected graphs.

Returns:

Diameter of the graph.

Return type:

float

Notes

Computed using NetworkX. See: https://en.wikipedia.org/wiki/Diameter_(graph_theory).

Warning

The graph must be connected. For disconnected graphs, NetworkX raises an exception unless a different definition (e.g. per component) is used.

property FiedlerVec

Fiedler value and Fiedler vector of the graph Laplacian.

The Fiedler value is the second smallest eigenvalue of the graph Laplacian, and the corresponding eigenvector is the Fiedler vector. These are commonly used in spectral graph theory to analyze graph connectivity and partitioning.

  • The Fiedler value (algebraic connectivity) indicates how well connected the graph is. Larger values imply stronger connectivity.

  • The Fiedler vector can be used for graph partitioning and clustering.

Returns:

float

Second smallest eigenvalue of the Laplacian matrix.

numpy.ndarray

Eigenvector corresponding to the Fiedler value.

Return type:

tuple of (float, numpy.ndarray)

Notes

Computed from the combinatorial Laplacian matrix using SciPy. See: https://en.wikipedia.org/wiki/Algebraic_connectivity.

Warning

Eigenvalues are not explicitly sorted. This implementation assumes that the second entry corresponds to the Fiedler value, which may not be correct in all cases. Eigenvalues should be sorted before selecting the Fiedler value.

property GlobalClusteringCoefficient

Global clustering coefficient of the graph.

The global clustering coefficient (also called transitivity) measures the fraction of closed triplets (triangles) over all connected triplets of nodes in the graph.

  • Value = 1 → every triplet is closed (maximal clustering, e.g., complete graph)

  • Value = 0 → no closed triplets (no clustering)

  • Intermediate values indicate the tendency of nodes to form triangles

Returns:

Global clustering coefficient in the range [0, 1].

Return type:

float

Notes

  • A triplet consists of three nodes connected by at least two edges.

  • Computed as: 3 × (number of triangles) / (number of connected triplets).

  • Returns 0.0 if the graph contains no triplets.

BracedVariants()

Generate all possible braced graph variants by combining face triangulations.

For each face in the graph, all possible triangulation scenarios are computed. A Cartesian product of these scenarios is then used to generate all global bracing combinations. Each combination produces a new graph variant with additional bracing edges.

Returns:

List of fully braced graph variants.

Return type:

list of Graph

Notes

  • Each face contributes a set of possible triangulation scenarios.

  • A global variant is created by selecting one scenario per face.

  • Newly added bracing edges are initialized with zero axial force.

  • Deep copies are used to avoid modifying the original graph state.

  • Requires itertools and copy.

LaplacianMatrix(normalized: bool = False, numpy: bool = False)

Return the Laplacian matrix of the graph.

The Laplacian matrix is a key operator in spectral graph theory.

  • Unnormalized (combinatorial Laplacian): L = D - A

  • Normalized Laplacian:

    L = I - D^{-1/2} A D^{-1/2}

Normalized Laplacians reduce degree bias and are widely used in spectral methods and graph neural networks.

See: https://en.wikipedia.org/wiki/Laplacian_matrix.

Parameters:

  • normalized (bool, optional) – If True, compute the normalized Laplacian. Default is False.

  • numpy (bool, optional) – If True, return result as a NumPy array. If False, return as a nested Python list. Default is False.

Returns:

Laplacian matrix of shape (n_nodes, n_nodes).

Return type:

numpy.ndarray or list of list of float

Notes

  • Handles isolated nodes safely in normalized form.

  • Uses degree and adjacency matrices internally.

static AxialForcesSolver_wo_reactions(truss, force_node_ids, ext_forces, sup_node_ids)

Solve axial forces in a truss using global equilibrium (matrix method), without explicitly forming a stiffness matrix.

The method assembles a global equilibrium system of the form:

A · x = -b

where: - A contains member direction cosines and support constraint equations - x contains unknown axial forces and support reactions - b contains externally applied nodal loads

A least-squares solution is used, allowing the method to handle statically determinate and indeterminate systems.

Parameters:

  • truss (Truss) – Truss object containing nodes and edges.

  • force_node_ids (list of int) – Node IDs where external forces are applied.

  • ext_forces (list of list[float]) – External force vectors at the corresponding nodes, given as [[Fx, Fy, Fz], …].

  • sup_node_ids (list of int) – Node IDs that are fully constrained (supports). Each support contributes three reaction components (x, y, z).

Returns:

  • X_forces (ndarray) – Axial forces in each member (length = number of edges). Positive/negative sign convention depends on member orientation.

  • R_forces (ndarray) – Reaction forces at supports (length = 3 × number of support nodes), ordered per node as [Rx, Ry, Rz].

Notes

  • The equilibrium matrix is constructed using direction cosines of each member.

  • Each node contributes three equilibrium equations (x, y, z).

  • Supports are modeled as unknown reaction forces (no displacement method used).

  • A least-squares solver (np.linalg.lstsq) is used:

    • Provides a solution even for over/underdetermined systems.

    • For well-posed statically determinate systems, the solution is exact.

  • Rows corresponding to unconstrained and unconnected DOFs are removed before solving to improve numerical stability.

  • Assumes pin-jointed truss behavior (axial forces only, no bending).

Raises:

  • ZeroDivisionError – If a member has zero length (coincident nodes).

  • ValueError – If input dimensions (nodes, forces) are inconsistent.

The method assembles a global equilibrium system:

A · x = -b

where: - A contains member direction cosines and selected support reaction components - x contains unknown axial forces and (active) support reactions - b contains externally applied nodal loads

This method computes the global stiffness matrix by assembling individual element stiffness matrices into a single system matrix using the direct stiffness method.

Each element contributes to the global matrix based on its connectivity and degrees of freedom (DOFs). The formulation supports both 2D and 3D truss systems.

Parameters:

graph (Graph) – Structural graph containing nodes, elements, geometry, and element stiffness definitions.

Returns:

Global stiffness matrix of size (dof * number_of_nodes, dof * number_of_nodes).

Return type:

ndarray

Notes

  • The stiffness matrix is assembled using:

    K = Σ (k_e * K_e)

    where: * k_e = (A * E) / L is the axial stiffness of each member * K_e is the element stiffness matrix in global coordinates

  • Degrees of freedom (DOFs) are determined automatically:

    • 2 DOFs per node → planar truss (X, Z)

    • 3 DOFs per node → spatial truss (X, Y, Z)

  • The method assumes:

    • Linear elastic material behavior

    • Pin-jointed truss members (axial forces only)

  • Assembly is performed by mapping local element DOFs to global DOF indices.

Raises:

  • ZeroDivisionError – If an element has zero length.

Reduce the global stiffness matrix by removing constrained DOFs.

This method constructs the reduced stiffness matrix by eliminating rows and columns associated with constrained (support) degrees of freedom. The resulting matrix corresponds only to the free DOFs and is used to solve the reduced equilibrium system.

Parameters:

  • K (ndarray) – Global stiffness matrix of size (ndof, ndof).

  • free_dofs (list of int) – Indices of free (unconstrained) degrees of freedom.

Returns:

Reduced stiffness matrix corresponding to free DOFs.

Return type:

ndarray

Notes

  • The reduction follows the standard FEM partitioning:

    [K_ff K_fc] [U_f] = [F_f] [K_cf K_cc] [U_c] [F_c]

    where constrained displacements U_c are known (typically zero), allowing the system to be reduced to:

    K_ff * U_f = F_f

Compute the redundancy matrix of a truss structure.

This method evaluates the redundancy (force indeterminacy) of a truss by constructing a compatibility matrix for redundant members and projecting it through the inverse of the reduced stiffness matrix. The resulting redundancy matrix quantifies the interaction between redundant forces.

Parameters:

  • truss (Graph) – The structural graph representing the truss system.

  • support_node_ids (list of int) – Node IDs corresponding to supports (fully constrained nodes).

Returns:

Return type:

ndarray or None

Notes

    r = m - n_free_dof where: m = number of members (edges) n_free_dof = number of unconstrained degrees of freedom

  • A simplified compatibility matrix B is constructed by selecting the last r members as redundants.

  • The redundancy matrix is computed as:

    R = B^T * K_ff^{-1} * B where K_ff is the reduced stiffness matrix.

  • This formulation is related to the force method in structural analysis.

  • Assumptions

    • The truss is stable and K_ff is invertible.

    • Redundant members are chosen heuristically (last r edges), which may not be optimal for all structures.

    • Compatibility matrix B is constructed in a simplified manner.

Perform form finding of a cable network using Schek’s Force Density Method (FDM).

This method computes the equilibrium geometry of a network by solving a linear system based on prescribed force densities. The structure is assumed to consist of axial members only (e.g., cables), and equilibrium is enforced at all free nodes.

  • axial_forces (ndarray) – Axial forces in each member (size = number of edges).

Export a graph to an enriched JSON file.

The method serializes a graph into a structured JSON format containing nodes, edges, loads, and faces. Optionally, additional analysis data (e.g. structural metrics, displacements, and buckling information) can be included.

Parameters:

  • graph (Graph) – Graph object to be exported.

  • folder_path (str) – Destination folder where the JSON file will be saved.

  • A (float or array-like) – Cross-sectional area(s) used for structural calculations.

  • density (float) – Material density used for mass computation.

  • graph_id (int, optional) – Identifier assigned to the graph in the exported dataset. Default is 0.

  • analysis_data (bool, optional) – If True, includes additional structural analysis results such as: - Euler buckling limits - Node displacements - Face angle information - Mass and geometric summaries

Return type:

None

Notes

The exported JSON includes the following primary sections:

  • edges: connectivity and force information

  • nodes: geometric and structural node properties

  • loads: applied forces

  • faces: topological face descriptions

When analysis_data=True, additional fields are appended:

  • Edge-level: Euler buckling limits

  • Node-level: displacements and face angle data

  • Global-level: mass, min/max length, and units dictionary

Warning

  • This function overwrites existing JSON files with the same name.

  • Assumes that graph object contains precomputed analysis attributes when analysis_data=True.

static ToJSON(graph, folder_path, graph_id=0)

Export a graph to a JSON file.

The method serializes a graph into a lightweight JSON format containing nodes, edges, and loads.

Parameters:

  • graph (Graph) – Graph object to be exported.

  • folder_path (str) – Destination folder where the JSON file will be saved.

  • graph_id (int, optional) – Identifier used in the output filename. Default is 0.

Return type:

None

Notes

The exported JSON contains:

  • edges: connectivity and axial force values

  • nodes: vertex coordinates (X, Y, Z)

  • loads: applied forces with anchor node and labels

This is a lightweight export and does not include derived structural or analysis data.
