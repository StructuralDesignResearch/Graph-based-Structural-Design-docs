static Solver(graph, force_node_ids, ext_forces, fixed_node_ids, q)

Perform form finding of a cable network using Schek’s Force Density Method (FDM).

This method computes the equilibrium geometry of a network by solving a linear system based on prescribed force densities. The structure is assumed to consist of axial members only (e.g., cables), and equilibrium is enforced at all free nodes.

Parameters:

  • graph (Graph) – Input graph representing the structural network. Must provide: - node coordinates (VertexObjs) - connectivity (EdgeNodePairs) - incidence matrix (Incidence)

  • force_node_ids (list of int) – Node IDs where external forces are applied.

  • ext_forces (list of list of float) – External force vectors applied at force_node_ids, given as [[Fx, Fy, Fz], ...].

  • fixed_node_ids (list of int) – Node IDs corresponding to fixed (support) nodes with prescribed coordinates.

  • q (array_like) –

    Force densities for each edge, defined as:

    q_i = F_i / L_i

    where F_i is the axial force and L_i is the member length.

Returns:

  • graph_ (Graph) – Updated graph with equilibrium node coordinates and computed axial forces.

  • axial_forces (ndarray) – Axial forces in each member (size = number of edges).

  • coords_ (ndarray) – Updated nodal coordinates of shape (n_nodes, 3).

Notes

  • The method follows the formulation by :contentReference[oaicite:0]{index=0}.

  • The equilibrium system is solved in the form:

D x = p - D_f x_f

where:

  • D = C^T Q C is the system matrix,

  • C is the reduced incidence matrix,

  • Q is the diagonal matrix of force densities,

  • x are unknown free node coordinates,

  • x_f are fixed node coordinates.

  • The method assumes:

    • Linear axial behavior

    • No bending stiffness

    • Static equilibrium

  • Suitable for:

    • Cable nets

    • Tension structures

    • Form-finding of lightweight systems

References

Schek, H.J. (1974). The Force Density Method for Form Finding and Computation of General Networks. Computer Methods in Applied Mechanics and Engineering.

static Edges(n, include_diagonals=False, orientation=None)

Generate edge list for an n x n grid graph.

Nodes are assumed to be indexed row-wise from left to right, top to bottom.

Parameters:

  • n (int) – Size of the grid (n x n nodes).

  • include_diagonals (bool, optional) – If True, diagonal edges are included. Default is False.

  • orientation (str, optional) –

    Direction of diagonal connections. Options:

    • ”BOTH”: include both diagonal directions

    • ”LR”: left-to-right diagonals only (up-right)

    • ”RL”: right-to-left diagonals only (up-left)

    Only used if include_diagonals is True.

Returns:

Edge list where each edge is represented as [node_i, node_j].

Return type:

list of list of int

Notes

  • Nodes are indexed as: node = row * n + col

  • Horizontal, vertical, and optional diagonal connections are generated depending on input parameters.

  • The graph is undirected; edges are returned in a single direction.

static Nodes(nodes, x_dist, y_dist)

Generate node coordinates for a regular 2D grid.

Nodes are arranged in a structured grid with uniform spacing in the x and y directions. All nodes lie in the XY plane (z = 0).

Parameters:

  • nodes (int) – Number of nodes per grid dimension (creates a nodes x nodes grid).

  • x_dist (float) – Spacing between adjacent nodes in the x-direction.

  • y_dist (float) – Spacing between adjacent nodes in the y-direction.

Returns:

List of node coordinates in the form [x, y, z], where z = 0 for all nodes.

Return type:

list of list of float

Notes

  • Nodes are generated in row-major order (y outer loop, x inner loop).

  • Total number of nodes is nodes².

  • The resulting grid is planar in the XY plane.
