Face

Core data structures for representing structural graphs, including nodes, edges, connectivity, and loads.

class Face.Face(v_ids_seq, id, graph)

Bases: object

Initialize a face in the structural graph.

Parameters:

  • v_ids_seq (list of int) – Ordered sequence of vertex (node) indices defining the face. The order is typically cyclic and determines the boundary of the face (e.g., polygon).

  • id (int) – Unique identifier for the face.

  • graph (Graph) – Reference to the parent graph object to which this face belongs. This allows access to global graph data such as node coordinates, edges, and connectivity.

Notes

A face represents a closed region in the graph, typically defined by a cycle of vertices. In structural systems, faces can correspond to cells or panels formed by interconnected elements.

property Coords2D

2D coordinates of the face vertices.

Returns:

Dictionary mapping vertex IDs to their (x, y) coordinates, extracted from the parent graph for all vertices belonging to the face.

Return type:

dict of int to list of float

property Coords3D

3D coordinates of the face vertices.

Returns:

Dictionary mapping vertex IDs to their (x, y, z) coordinates, extracted from the parent graph for all vertices belonging to the face.

Return type:

dict of int to list of float

property Edges

Edge IDs defining the boundary of the face.

Returns:

Ordered edge identifiers corresponding to the face boundary. The ordering follows the vertex sequence of the face (typically counter-clockwise).

Return type:

list of int

property EdgeObjs

Edge objects defining the boundary of the face.

Returns:

Ordered edge objects forming the face boundary. The ordering follows the vertex sequence of the face (typically counter-clockwise).

Return type:

list of Edge

property Id

Unique identifier of the face.

Returns:

Identifier used to distinguish the face (cycle) within the graph.

Return type:

int

property Vertices

Ordered sequence of vertex IDs defining the face.

Returns:

Ordered vertex IDs forming the face boundary (typically in counter-clockwise order).

Return type:

list of int

property VertexObjs

Vertex objects defining the face boundary.

Returns:

Vertex objects corresponding to the vertices in the face, filtered from the parent graph and ordered according to the face vertex sequence.

Return type:

list of Vertex

TriangulationScenarios()

Enumerate all non-crossing triangulations of a simple polygon.

The method computes all valid triangulations of the face defined by its vertex sequence. Each triangulation represents a possible way to decompose the polygon into non-overlapping triangles, which can be interpreted as alternative structural bar configurations.

Returns:

List of triangulation scenarios. Each scenario is represented as a set of edges (vertex index pairs) defining the diagonals required for that triangulation.

Return type:

list of set of tuple of int

Notes

  • A valid input must contain at least 4 distinct vertices.

  • If fewer than 4 vertices are present, an empty list is returned.

  • The polygon is assumed to be simple (non-self-intersecting).

  • Duplicate closing vertex (first == last) is automatically removed.

  • Triangulations are non-crossing and exhaustive.

Raises:

No exceptions are raised. Invalid inputs return an empty list. –

Warning

This method prints a warning message for invalid or degenerate faces instead of raising errors.
