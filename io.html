static ByCSVpath(folder_path)

Load multiple graphs from CSV files in a directory.

The method reads graph, node, edge, and optionally load (force) information from CSV files and constructs a dictionary of Graph objects indexed by graph_id.

Expected files:

  • graphs.csv

  • edges.csv

  • nodes.csv

  • forces.csv (optional)

Parameters:

folder_path (str) – Path to the folder containing the CSV files. Platform-specific path separators are handled internally.

Returns:

Dictionary mapping each graph_id to a corresponding Graph instance.

Return type:

dict of int -> Graph

Notes

  • Nodes are expected to contain columns: X, Y, Z.

  • Edges are expected to contain: start_id, end_id, axial_f.

  • Forces (if provided) must contain: X, Y, Z, anchor_id, label.

  • If forces.csv is missing, graphs are created without load data.

  • The function assumes consistency between graph_id counts across nodes, edges, and graphs tables.

Warning

  • If CSV structure is inconsistent, the function will print an error message and return nothing.

  • Uses platform-specific path handling (Linux vs Windows).

static ToCSV(graphs, folder_path)

Export a list of graphs to CSV files.

The method serializes graph data into four CSV files:

  • graphs.csv: global graph-level features

  • edges.csv: edge connectivity and attributes

  • nodes.csv: node attributes and geometry

  • forces.csv: load/force data (if available)

Parameters:

  • graphs (iterable of Graph) – Collection of Graph objects to be exported.

  • folder_path (str) – Destination folder where CSV files will be written. Existing files with the same names will be overwritten.

Return type:

None

Notes

  • Each graph is assigned a sequential graph_id based on iteration order.

  • Graph-level features include structural metrics such as: planarity, static action, lengths, and force statistics.

  • Node coordinates are written as (X, Y, Z).

  • Edge list uses start/end node IDs and axial force values.

  • Force data is written only if available in the Graph objects.

  • Existing CSV files in the target directory are deleted before writing.

Warning

  • This function performs file deletion before writing; ensure no important data exists in the target folder.

  • No schema validation is performed on input graphs.

static ToJSONadv(graph, folder_path, A=1, density=1, graph_id=0, analysis_data=False)

Export a graph to an enriched JSON file.

The method serializes a graph into a structured JSON format containing nodes, edges, loads, and faces. Optionally, additional analysis data (e.g. structural metrics, displacements, and buckling information) can be included.

Parameters:

  • graph (Graph) – Graph object to be exported.

  • folder_path (str) – Destination folder where the JSON file will be saved.

  • A (float or array-like) – Cross-sectional area(s) used for structural calculations.

  • density (float) – Material density used for mass computation.

  • graph_id (int, optional) – Identifier assigned to the graph in the exported dataset. Default is 0.

  • analysis_data (bool, optional) – If True, includes additional structural analysis results such as: - Euler buckling limits - Node displacements - Face angle information - Mass and geometric summaries

Return type:

None

Notes

The exported JSON includes the following primary sections:

  • edges: connectivity and force information

  • nodes: geometric and structural node properties

  • loads: applied forces

  • faces: topological face descriptions

When analysis_data=True, additional fields are appended:

  • Edge-level: Euler buckling limits

  • Node-level: displacements and face angle data

  • Global-level: mass, min/max length, and units dictionary

Warning

  • This function overwrites existing JSON files with the same name.

  • Assumes that graph object contains precomputed analysis attributes when analysis_data=True.

static ToJSON(graph, folder_path, graph_id=0)

Export a graph to a JSON file.

The method serializes a graph into a lightweight JSON format containing nodes, edges, and loads.

Parameters:

  • graph (Graph) – Graph object to be exported.

  • folder_path (str) – Destination folder where the JSON file will be saved.

  • graph_id (int, optional) – Identifier used in the output filename. Default is 0.

Return type:

None

Notes

The exported JSON contains:

  • edges: connectivity and axial force values

  • nodes: vertex coordinates (X, Y, Z)

  • loads: applied forces with anchor node and labels

This is a lightweight export and does not include derived structural or analysis data.
