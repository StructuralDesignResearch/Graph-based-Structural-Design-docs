Graph

Core data structures for representing structural graphs, including nodes, edges, connectivity, and loads.

class Graph.Graph(nodal_coords, edges, weights)

Bases: object

Initialize the representation of a graph.

Parameters:

  • nodal_coords (array-like of shape (n_nodes, n_dims)) – Coordinates of each node in the structure. Each row corresponds to a node and contains its spatial coordinates.

  • edges (array-like or list of lists) – List of edges in the graph. Each edge is represented as a pair of node indices.

  • weights (array-like) – Edge weights representing axial forces.

  • loads (array-like, optional) – External load vectors applied to the nodes. Default is None.

  • fixed_nodes (list of int, optional) – Indices of nodes that are fixed (supports). These nodes are assigned a label of 0, while free nodes are assigned 1. If None, all nodes are initialized with label -1.

  • load_labels (array-like, optional) – Labels indicating how loads are applied to nodes. Used in combination with loads to construct force vectors.

Notes

This initializer constructs the full graph representation of a structural system, including geometry, topology, and optionally applied loads and boundary conditions.

property ClosenessCentrality

Closeness centrality of the graph.

Closeness centrality measures how close a node is to all other nodes in the graph based on shortest path distances. Nodes with higher values can reach all other nodes more quickly on average.

Higher values indicate nodes that are more centrally located in terms of graph distance.

Returns:

Mapping from node to closeness centrality value in the range [0, 1].

Return type:

dict

Notes

Computed using NetworkX closeness centrality. See: https://en.wikipedia.org/wiki/Closeness_centrality.

property Coords2D

2D or projected node coordinates of the graph.

The coordinates are automatically projected to a 2D plane if the input data lies in a coordinate plane (XY, XZ, or YZ). Otherwise, full 3D coordinates are returned.

Projection rules:

  • If all z = 0 → returns XY coordinates

  • If all y = 0 → returns XZ coordinates

  • If all x = 0 → returns YZ coordinates

  • Otherwise → returns full 3D coordinates

Returns:

Node coordinates in reduced 2D form if planar, otherwise full 3D coordinates. Shape is (n_nodes, 2) or (n_nodes, 3).

Return type:

list of list of float

property DegreeCentrality

Degree centrality of the graph.

Degree centrality measures the fraction of nodes a given node is directly connected to. It is the simplest form of centrality and reflects immediate connectivity in the network.

Nodes with high degree centrality are directly connected to many other nodes.

Returns:

Mapping from node to degree centrality value in the range [0, 1].

Return type:

dict

Notes

Computed using NetworkX degree centrality. See: https://en.wikipedia.org/wiki/Degree_centrality.

property Density

Graph density.

Graph density measures how close the graph is to being complete, defined as the ratio of existing edges to the maximum possible number of edges.

  • Density = 0 → no edges in the graph

  • Density = 1 → fully connected (complete graph)

  • Intermediate values indicate how sparsely or densely connected the graph is

Returns:

Graph density in the range [0, 1].

Return type:

float

Notes

Computed using NetworkX. See: https://networkx.org/documentation/stable/reference/generated/networkx.density.html

property Diameter

Graph diameter.

The diameter of a graph is the longest shortest-path distance between any pair of nodes. It represents the greatest minimal distance required to travel between two nodes in the network.

  • The diameter represents the worst-case shortest-path distance in the graph.

  • It provides a measure of the “size” of the graph in terms of connectivity.

  • Larger values indicate more spread-out or less tightly connected graphs.

Returns:

Diameter of the graph.

Return type:

float

Notes

Computed using NetworkX. See: https://en.wikipedia.org/wiki/Diameter_(graph_theory).

Warning

The graph must be connected. For disconnected graphs, NetworkX raises an exception unless a different definition (e.g. per component) is used.

property FiedlerVec

Fiedler value and Fiedler vector of the graph Laplacian.

The Fiedler value is the second smallest eigenvalue of the graph Laplacian, and the corresponding eigenvector is the Fiedler vector. These are commonly used in spectral graph theory to analyze graph connectivity and partitioning.

  • The Fiedler value (algebraic connectivity) indicates how well connected the graph is. Larger values imply stronger connectivity.

  • The Fiedler vector can be used for graph partitioning and clustering.

Returns:

float

Second smallest eigenvalue of the Laplacian matrix.

numpy.ndarray

Eigenvector corresponding to the Fiedler value.

Return type:

tuple of (float, numpy.ndarray)

Notes

Computed from the combinatorial Laplacian matrix using SciPy. See: https://en.wikipedia.org/wiki/Algebraic_connectivity.

Warning

Eigenvalues are not explicitly sorted. This implementation assumes that the second entry corresponds to the Fiedler value, which may not be correct in all cases. Eigenvalues should be sorted before selecting the Fiedler value.

property GlobalClusteringCoefficient

Global clustering coefficient of the graph.

The global clustering coefficient (also called transitivity) measures the fraction of closed triplets (triangles) over all connected triplets of nodes in the graph.

  • Value = 1 → every triplet is closed (maximal clustering, e.g., complete graph)

  • Value = 0 → no closed triplets (no clustering)

  • Intermediate values indicate the tendency of nodes to form triangles

Returns:

Global clustering coefficient in the range [0, 1].

Return type:

float

Notes

  • A triplet consists of three nodes connected by at least two edges.

  • Computed as: 3 × (number of triangles) / (number of connected triplets).

  • Returns 0.0 if the graph contains no triplets.

Return type:

property IsGeometricallyPlanar

Check whether the graph is geometrically planar.

A graph is considered planar here if no two non-adjacent edges intersect in the embedding defined by the node coordinates.

Returns:

True if the graph has no edge intersections (excluding adjacent edges), False otherwise.

Return type:

bool

Notes

  • Planarity is evaluated based on the current geometric embedding of the graph, not purely on graph topology.

  • Intersection detection is performed using segment–segment intersection tests.

Warning

This check depends on node coordinates and may differ from topological planarity (i.e., whether the graph can be drawn planar without crossings).

BetweennessCentralityNodes(use_weights=True)

Compute the betweenness centrality of graph nodes.

Betweenness centrality measures how often a node appears on the shortest paths between all pairs of other nodes in a graph. It quantifies the importance of a node in facilitating flow through the network.

  • Nodes with high betweenness act as bridges or bottlenecks in the graph.

  • They may control or mediate interactions between different regions.

  • Useful for identifying critical nodes in communication, transport, or structural vulnerability analysis.

Parameters:

use_weights (bool, optional) – If True, edge weights (attribute ‘weight’) are used when computing shortest paths. If False, all edges are treated as unweighted. Default is True.

Returns:

Dictionary mapping nodes to their betweenness centrality values.

Return type:

dict

Notes

  • Computation is performed using NetworkX.

  • Values are normalized to the range [0, 1].

  • See: https://en.wikipedia.org/wiki/Betweenness_centrality for details.

BracedVariants()

Generate all possible braced graph variants by combining face triangulations.

For each face in the graph, all possible triangulation scenarios are computed. A Cartesian product of these scenarios is then used to generate all global bracing combinations. Each combination produces a new graph variant with additional bracing edges.

Returns:

List of fully braced graph variants.

Return type:

list of Graph

Notes

  • Each face contributes a set of possible triangulation scenarios.

  • A global variant is created by selecting one scenario per face.

  • Newly added bracing edges are initialized with zero axial force.

  • Deep copies are used to avoid modifying the original graph state.

  • Requires itertools and copy.

FaceAngles()

Compute node angles for all internal faces of a planar graph (CCW ordered).

For each node, the method computes the interior angles contributed by adjacent faces and sorts them in counter-clockwise order based on edge direction.

Returns:

  • node_angles (dict) – Mapping {node: [angles]} where angles are sorted CCW around each node.

  • node_angles_face_ids (dict) – Mapping {node: [face_ids]} corresponding to the same CCW ordering.

  • angle_map (dict) – Mapping {(node, face_id): angle} for direct lookup of angles.

  • faces (list of list) – List of faces (cycles) as ordered node sequences.

Notes

  • Faces are obtained from self.Cycles().

  • Angles are computed using vector-based geometry.

  • Sorting is based on the polar angle of outgoing edges.

LaplacianMatrix(normalized: bool = False, numpy: bool = False)

Return the Laplacian matrix of the graph.

The Laplacian matrix is a key operator in spectral graph theory.

  • Unnormalized (combinatorial Laplacian): L = D - A

  • Normalized Laplacian:

    L = I - D^{-1/2} A D^{-1/2}

Normalized Laplacians reduce degree bias and are widely used in spectral methods and graph neural networks.

See: https://en.wikipedia.org/wiki/Laplacian_matrix.

Parameters:

  • normalized (bool, optional) – If True, compute the normalized Laplacian. Default is False.

  • numpy (bool, optional) – If True, return result as a NumPy array. If False, return as a nested Python list. Default is False.

Returns:

Laplacian matrix of shape (n_nodes, n_nodes).

Return type:

numpy.ndarray or list of list of float

Notes

  • Handles isolated nodes safely in normalized form.

  • Uses degree and adjacency matrices internally.

MergeNodesById(node_id, to_node_id)

Merge two adjacent and unconstrained nodes in the graph.

The method removes node_id and redirects all its incident edges to to_node_id, updating the connectivity, vertices, edges, and adjacency structures accordingly.

Parameters:

  • node_id (int) – ID of the node to be merged (removed).

  • to_node_id (int) – ID of the target node that absorbs the connectivity.

Returns:

Updated graph instance if merge is successful, otherwise None.

Return type:

Graph or None

Notes

  • Nodes must be adjacent.

  • Nodes must be unconstrained according to Graph.UnconstrainedNodes.

  • All internal IDs are relabeled after modification.

MirrorGraph(plane_origin, plane_normal)

Generate a mirrored version of the graph across a given plane.

The method constructs a symmetric graph by reflecting nodes and edges across a specified plane while handling boundary conditions such as: - nodes lying on the symmetry plane - nodes with valency 1 (removed from duplication) - edges lying along the symmetry axis (invalid configuration)

Parameters:

  • plane_origin (tuple (x, y, z)) – A point on the symmetry plane.

  • plane_normal (tuple (nx, ny, nz)) – Normal vector defining the symmetry plane.

Returns:

A new mirrored Graph instance if successful, otherwise None.

Return type:

Graph or None

Notes

  • Nodes on the symmetry plane are treated specially:

    • Valency 1 → removed

    • Others → not duplicated

  • The graph is duplicated and reflected using a node mapping.

PageRank(alpha=0.85)

Compute the PageRank of the graph nodes.

PageRank assigns higher scores to nodes that are linked to by many important nodes, capturing a recursive notion of influence.

See: https://en.wikipedia.org/wiki/PageRank.

Parameters:

alpha (float, optional) – Damping factor representing the probability of continuing a random walk. Default is 0.85.

Returns:

PageRank scores for each node in the graph.

Return type:

list of float

Notes

  • Computed using NetworkX implementation.

  • Output order follows NetworkX node ordering.
