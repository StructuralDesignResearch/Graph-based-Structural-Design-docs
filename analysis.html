static AxialForcesSolver_wo_reactions(truss, force_node_ids, ext_forces, sup_node_ids)

Solve axial forces in a truss using global equilibrium (matrix method), without explicitly forming a stiffness matrix.

The method assembles a global equilibrium system of the form:

A · x = -b

where: - A contains member direction cosines and support constraint equations - x contains unknown axial forces and support reactions - b contains externally applied nodal loads

A least-squares solution is used, allowing the method to handle statically determinate and indeterminate systems.

Parameters:

  • truss (Truss) – Truss object containing nodes and edges.

  • force_node_ids (list of int) – Node IDs where external forces are applied.

  • ext_forces (list of list[float]) – External force vectors at the corresponding nodes, given as [[Fx, Fy, Fz], …].

  • sup_node_ids (list of int) – Node IDs that are fully constrained (supports). Each support contributes three reaction components (x, y, z).

Returns:

  • X_forces (ndarray) – Axial forces in each member (length = number of edges). Positive/negative sign convention depends on member orientation.

  • R_forces (ndarray) – Reaction forces at supports (length = 3 × number of support nodes), ordered per node as [Rx, Ry, Rz].

Notes

  • The equilibrium matrix is constructed using direction cosines of each member.

  • Each node contributes three equilibrium equations (x, y, z).

  • Supports are modeled as unknown reaction forces (no displacement method used).

  • A least-squares solver (np.linalg.lstsq) is used:

    • Provides a solution even for over/underdetermined systems.

    • For well-posed statically determinate systems, the solution is exact.

  • Rows corresponding to unconstrained and unconnected DOFs are removed before solving to improve numerical stability.

  • Assumes pin-jointed truss behavior (axial forces only, no bending).

Raises:

  • ZeroDivisionError – If a member has zero length (coincident nodes).

  • ValueError – If input dimensions (nodes, forces) are inconsistent.

static AxialForcesSolver_w_reactions(truss, force_node_ids, ext_forces, sup_node_ids, sup_forces)

Solve axial forces in a truss using global equilibrium with partially prescribed support reactions.

The method assembles a global equilibrium system:

A · x = -b

where: - A contains member direction cosines and selected support reaction components - x contains unknown axial forces and (active) support reactions - b contains externally applied nodal loads

Unlike the least-squares version, this formulation removes inactive degrees of freedom and solves the reduced system directly using np.linalg.solve.

Parameters:

  • truss (Truss) – Truss object containing nodes and edges.

  • force_node_ids (list of int) – Node IDs where external forces are applied.

  • ext_forces (list of list[float]) – External force vectors at the corresponding nodes, given as [[Fx, Fy, Fz], …].

  • sup_node_ids (list of int) – Node IDs that act as supports.

  • sup_forces (list of list[float]) – Support reaction activation per node, given as [[Rx, Ry, Rz], …]. Each component acts as a binary selector: - nonzero → reaction component is included as an unknown - zero → reaction component is excluded from the system

Returns:

Flattened solution vector containing: - axial forces (first E entries) - active support reaction components (remaining entries)

Return type:

ndarray

Notes

  • The equilibrium matrix is constructed using direction cosines of each member.

  • Each node contributes three equilibrium equations (x, y, z).

  • Support reactions are selectively included based on sup_forces.

  • Rows corresponding to unconnected DOFs and columns corresponding to inactive unknowns are removed before solving.

  • The system is solved using np.linalg.solve, therefore:

    • The reduced system must be square and non-singular.

    • This method assumes a well-posed, fully constrained structure.

  • Assumes pin-jointed truss behavior (axial forces only).

Raises:

  • LinAlgError – If the system matrix is singular or not square.

  • ZeroDivisionError – If a member has zero length (coincident nodes).

  • ValueError – If input dimensions are inconsistent.

Analysis(graph, support_node_ids)

Perform structural analysis of a truss using the stiffness method.

This method computes nodal displacements, member axial forces, strains, and stresses by assembling and solving the global equilibrium equations of the system.

The analysis pipeline consists of:

  1. Dimensionality reduction (2D/3D handling and DOF definition)

  2. Global stiffness matrix assembly

  3. Application of boundary conditions (supports)

  4. Reduction of the system to free degrees of freedom

  5. Solution of nodal displacements

  6. Recovery of full displacement vector

  7. Computation of member forces, strains, and stresses

  8. Update of the graph with analysis results

Parameters:

  • graph (Graph) – Structural graph representing the truss system, including geometry, connectivity, and material properties.

  • support_node_ids (list of int) – Node IDs corresponding to supports (fully constrained nodes).

Returns:

  • axial_forces (list of float) – Axial force in each member.

  • disp (list of list[float]) – Nodal displacements for each node as [Ux, Uy, Uz].

  • strain (list of float) – Axial strain in each member.

  • stress (list of float) – Axial stress in each member.

Notes

  • Uses the linear elastic stiffness method (small displacement assumption).

  • Degrees of freedom (DOFs) are automatically determined based on the dimensionality of the structure.

  • Supports are applied by removing constrained DOFs from the system.

  • The reduced system is solved using standard linear algebra solvers.

  • Results are written back to the input graph via Graph.UpdateAnalyzedGraph.

Raises:

  • LinAlgError – If the reduced stiffness matrix is singular (mechanism or insufficient constraints).

  • ValueError – If input data (geometry, supports, loads) is inconsistent.

GlobalForceVector(truss, support_node_ids, dof=2)

Assemble the global force vector for a truss structure.

This method constructs the nodal force vector by mapping externally applied loads to the corresponding degrees of freedom (DOFs) of each node. Loads applied at support nodes are ignored, as those DOFs are assumed to be constrained.

Parameters:

  • truss (Graph) – Structural graph containing nodes and applied loads (LoadObjs).

  • support_node_ids (list of int) – Node IDs corresponding to supports (constrained nodes).

  • dof (int, optional) – Number of degrees of freedom per node: - 2 → planar analysis (X, Z) - 3 → spatial analysis (X, Y, Z) Default is 2.

Returns:

Global force vector of shape (dof * number_of_nodes, 1), where forces are arranged per node as: [Fx, Fz] (2D) or [Fx, Fy, Fz] (3D).

Return type:

ndarray

Notes

  • Loads are extracted from truss.LoadObjs.

  • For 2D problems, only X and Z components are considered.

  • For 3D problems, X, Y, and Z components are included.

  • Forces applied at support nodes are excluded, since their displacements are constrained and handled separately.

Raises:

ValueError – If dof is not equal to 2 or 3.

GlobalStiffnessMatrix(truss)

Assemble the global stiffness matrix of a truss structure.

This method computes the global stiffness matrix by assembling individual element stiffness matrices into a single system matrix using the direct stiffness method.

Each element contributes to the global matrix based on its connectivity and degrees of freedom (DOFs). The formulation supports both 2D and 3D truss systems.

Parameters:

truss (Graph) – Structural graph containing nodes, elements, geometry, and element stiffness definitions.

Returns:

Global stiffness matrix of size (dof * number_of_nodes, dof * number_of_nodes).

Return type:

ndarray

Notes

  • The stiffness matrix is assembled using:

    K = Σ (k_e * K_e) where: * k_e = (A * E) / L is the axial stiffness of each member * K_e is the element stiffness matrix in global coordinates

  • Degrees of freedom (DOFs) are determined automatically:

    • 2 DOFs per node → planar truss (X, Z)

    • 3 DOFs per node → spatial truss (X, Y, Z)

  • The method assumes:

    • Linear elastic material behavior

    • Pin-jointed truss members (axial forces only)

  • Assembly is performed by mapping local element DOFs to global DOF indices.

Raises:

  • ZeroDivisionError – If an element has zero length.

  • ValueError – If the dimensionality (DOF) is not supported.

RecoverGlobalDisplacement(Uf, free_dofs, ndof)

Reconstruct the full global displacement vector from reduced DOFs.

This method expands the displacement solution obtained from the reduced system (free degrees of freedom only) back into the full global displacement vector. Constrained DOFs (e.g., supports) are assigned zero displacement.

Parameters:

  • Uf (ndarray) – Displacement vector corresponding to the free degrees of freedom.

  • free_dofs (list of int) – Indices of the free degrees of freedom in the global system.

  • ndof (int) – Total number of degrees of freedom in the full system (typically dof × number_of_nodes).

Returns:

Global displacement vector of length ndof, where: - entries corresponding to free DOFs are filled from Uf - constrained DOFs are set to zero

Return type:

ndarray

Notes

  • This step is part of the standard stiffness method workflow:

    1. Reduce system → solve for free DOFs

    2. Recover full displacement vector

  • Assumes that all constrained DOFs have zero displacement (Dirichlet boundary conditions).

  • The ordering of Uf must match the ordering of free_dofs.

Raises:

ValueError – If the length of Uf does not match the number of free_dofs.

ReduceForceSupports(F, free_dofs)

Extract the reduced force vector corresponding to free DOFs.

This method filters the global force vector to retain only the entries associated with free (unconstrained) degrees of freedom. It is used when forming the reduced system of equations after applying support constraints.

Parameters:

  • F (ndarray) – Global force vector of size (ndof, 1) or (ndof,).

  • free_dofs (list of int) – Indices of the free degrees of freedom.

Returns:

Reduced force vector containing only the entries of F corresponding to free_dofs.

Return type:

ndarray

Notes

  • This operation is equivalent to selecting rows of the global system associated with unconstrained DOFs.

  • Typically used together with a reduced stiffness matrix:

    K_ff * U_f = F_f

  • Assumes that free_dofs are valid indices of F.

ReduceStiffnessSupports(K, support_dofs)

Reduce the global stiffness matrix by removing constrained DOFs.

This method constructs the reduced stiffness matrix by eliminating rows and columns associated with constrained (support) degrees of freedom. The resulting matrix corresponds only to the free DOFs and is used to solve the reduced equilibrium system.

Parameters:

  • K (ndarray) – Global stiffness matrix of size (ndof, ndof).

  • support_dofs (list of int) – Indices of constrained (support) degrees of freedom.

Returns:

K_ffndarray

Reduced stiffness matrix corresponding to free DOFs.

free_dofslist of int

Indices of the free (unconstrained) DOFs.

Return type:

tuple (K_ff, free_dofs)

Notes

  • The reduction follows the standard FEM partitioning:

    [K_ff K_fc] [U_f] = [F_f] [K_cf K_cc] [U_c] [F_c]

    where constrained displacements U_c are known (typically zero), allowing the system to be reduced to:

    K_ff * U_f = F_f

  • Uses NumPy advanced indexing (np.ix_) to extract the submatrix.

  • The ordering of DOFs is preserved.

RedundancyMatrix(truss, reduced_coords, support_dofs)

Compute the redundancy matrix of a truss structure.

This method evaluates the redundancy (force indeterminacy) of a truss by constructing a compatibility matrix for redundant members and projecting it through the inverse of the reduced stiffness matrix. The resulting redundancy matrix quantifies the interaction between redundant forces.

Parameters:

  • truss (Graph) – The structural graph representing the truss system.

  • reduced_coords (ndarray) – Reduced nodal coordinates (after dimensionality reduction), used to determine the number of DOFs per node.

  • support_dofs (list of int) – Indices of constrained (support) degrees of freedom.

Returns:

Redundancy matrix of size (r, r), where r is the number of redundant members. Returns None if the truss is statically determinate (i.e., no redundant members).

Return type:

ndarray or None

Notes

  • The number of redundant members is estimated as:

    r = m - n_free_dof where: m = number of members (edges) n_free_dof = number of unconstrained degrees of freedom

  • A simplified compatibility matrix B is constructed by selecting the last r members as redundants.

  • The redundancy matrix is computed as:

    R = B^T * K_ff^{-1} * B where K_ff is the reduced stiffness matrix.

  • This formulation is related to the force method in structural analysis.

Assumptions

  • The truss is stable and K_ff is invertible.

  • Redundant members are chosen heuristically (last r edges), which may not be optimal for all structures.

  • Compatibility matrix B is constructed in a simplified manner.

RigidityMatrix(truss, reduced_coords)

Compute the rigidity (compatibility) matrix of a truss structure.

The rigidity matrix relates nodal displacements to axial member elongations. Each row corresponds to a bar (edge), and each column corresponds to a degree of freedom of a node.

Parameters:

  • truss (Graph) – The structural graph representing the truss system.

  • reduced_coords (ndarray of shape (n_nodes, d)) – Nodal coordinates after dimensionality reduction (e.g., removal of zero components). The dimension d is typically 2 or 3.

Returns:

Rigidity matrix of shape (n_bars, d * n_nodes), where each row encodes the directional contribution of a bar to nodal DOFs.

Return type:

ndarray

Notes

  • Each row corresponds to a bar connecting nodes i and j.

  • For a bar (i, j), the matrix entries are constructed using the unit direction vector:

    u_ij = (x_i - x_j) / ||x_i - x_j||

and inserted as:

R[k, d*i : d*(i+1)] = u_ij R[k, d*j : d*(j+1)] = -u_ij

  • The rigidity matrix maps nodal displacements u to member extensions e:

    e = R * u

  • This matrix is closely related to:

    • the equilibrium matrix (its transpose)

    • stiffness matrix formulation in finite element methods

    • rigidity theory in structural mechanics and graph theory

  • A zero-length bar is invalid and will raise an error.

References

https://mathworld.wolfram.com/RigidityMatrix.html

SolveDisplacements(Kff, Ff)

Solve the linear system for unknown nodal displacements at free degrees of freedom.

This function computes the displacement vector of the unconstrained (free) degrees of freedom in a structural system by solving the reduced stiffness equation:

Kff * Uf = Ff

where: Kff is the stiffness matrix restricted to free DOFs, Ff is the corresponding reduced external force vector, Uf is the unknown displacement vector.

Parameters:

  • Kff (ndarray) – Reduced stiffness matrix corresponding to free degrees of freedom. Shape: (n_free_dofs, n_free_dofs).

  • Ff (ndarray) – Reduced external force vector corresponding to free DOFs. Shape: (n_free_dofs,) or (n_free_dofs, 1).

Returns:

Displacement vector of free degrees of freedom. Shape: (n_free_dofs,).

Return type:

ndarray

Notes

  • Assumes the system is linear elastic and in static equilibrium.

  • Requires Kff to be non-singular; otherwise the system is unstable or under-constrained.

  • Uses a direct solver (NumPy LU-based solver via np.linalg.solve), which is efficient for dense systems but not optimal for large sparse structures.

Raises:

LinAlgError – If Kff is singular or ill-conditioned.

See also

numpy.linalg.solve

underlying linear solver used.

SolveMemberForces(truss, coords, plane, Ug)

Compute internal axial response of each truss member based on global displacements.

This function evaluates element-level mechanical quantities (strain, stress, and axial force) by projecting the relative nodal displacements of each bar onto its axial direction.

For each element (i, j), the method: - Computes the element length and unit direction vector - Extracts nodal displacements from the global displacement vector Ug - Projects relative displacement onto the element axis - Computes axial strain, stress, and force using linear elastic relations

The results are stored internally in the object (self.axialforces, self.strains, self.stresses, self.disp) and are also returned as a structured list.

Parameters:

  • truss (Graph-like object) – Structure containing node connectivity (EdgeNodePairs) and element data.

  • coords (ndarray) – Node coordinate array of shape (n_nodes, dim), where dim ∈ {2, 3}.

  • plane (str) – Projection plane used for displacement storage (“XY”, “XZ”, “YZ”, “XYZ”).

  • Ug (ndarray) – Global displacement vector containing all nodal DOFs.

Returns:

Element-wise results containing: - element : tuple (i, j) - length : float - direction : list[float] unit vector from i to j - u_i : list[float] nodal displacement at node i - u_j : list[float] nodal displacement at node j - delta_u_axial : float axial relative displacement - strain : float engineering strain - stress : float axial stress (E * strain) - force : float axial force (A * stress, positive = tension)

Return type:

list of dict

Notes

  • Assumes small deformation linear elasticity.

  • Positive axial force corresponds to tension, negative to compression.

  • Updates internal state dictionaries: self.disp, self.strains, self.stresses, self.axialforces.

static ApplyCrossSectionCalcVolume(forces, lengths, cross_sections, sigma_allowed)

Assign the minimum feasible cross-sectional area to each truss member based on axial force demand, and compute the resulting total structural volume.

For each member, the required stress demand is evaluated and the smallest available cross-section that satisfies the allowable stress constraint is selected. The total structural volume is then computed as the sum of each member’s volume contribution.

Parameters:

  • forces (list[float]) – Axial forces in each member (N). Positive or negative values are handled via absolute value.

  • lengths (list[float]) – Member lengths (m), corresponding one-to-one with forces.

  • cross_sections (list[float]) – Available cross-sectional areas (mm²), assumed sorted in ascending order. The algorithm selects the smallest section that satisfies the stress constraint.

  • sigma_allowed (float) – Allowable material stress (MPa = N/mm²).

Returns:

  • assigned_sections (list[float]) – Selected cross-sectional area for each member (mm²).

  • total_volume (float) – Total structural volume (m³), computed as sum(A * L) with unit conversion.

Raises:

ValueError – If input lists have inconsistent lengths or if no feasible cross-section exists for a given force demand.

Notes

  • Stress check: |F| / A ≤ σ_allowable

  • Unit consistency:

    • Force: N

    • Area: mm² (converted internally to m² for volume)

    • Length: m

  • This is a greedy per-member optimal sizing strategy, not a global optimization.

static ReduceDimensionsDOF(coords)

Detect whether a truss geometry is planar or fully 3D and reduce its coordinate representation accordingly.

The function inspects nodal coordinates and determines if all nodes lie on one of the principal planes (XY, XZ, or YZ). If so, it removes the redundant coordinate direction and returns a reduced 2D system. Otherwise, the full 3D representation is preserved.

This is used to simplify subsequent structural analysis by reducing the number of degrees of freedom when the structure is planar.

Parameters:

coords (array_like) – Nodal coordinates as an (N x 3) array or list, where each row is (x, y, z).

Returns:

  • reduced_coords (ndarray) – Coordinate array with reduced dimensionality: - (N x 2) if the structure is planar - (N x 3) if the structure is fully 3D

  • dof (int) – Number of degrees of freedom per node: - 2 for planar structures - 3 for spatial structures

  • plane (str) – Identified geometric plane or space: - “XY”, “XZ”, “YZ” for planar cases - “XYZ” for full 3D structures

Notes

  • Planarity is detected using np.allclose against zero coordinate components.

  • Intended for preprocessing in finite element truss analysis pipelines.

static TributaryForces(coords, node_ids, load_per_meter)

Compute equivalent nodal forces from a uniformly distributed line load using tributary lengths.

This method approximates how a distributed load along a polyline (e.g., beam or truss chord) is converted into equivalent nodal point loads. Each internal node receives a force based on half the adjacent segment lengths on either side, while end nodes receive half-contributions.

Parameters:

  • coords (array-like) – Nodal coordinates, where each entry is assumed to contain at least an x-coordinate used for spacing (coords[i][0]).

  • node_ids (list[int]) – Ordered list of node indices defining the loaded polyline path.

  • load_per_meter (float) – Uniform distributed load intensity (force per unit length).

Returns:

Mapping of node_id → equivalent nodal force (same force units as load_per_meter × length).

Return type:

dict[int, float]

Notes

  • The method assumes nodes are ordered along a 1D projection (uses x-coordinate spacing).

  • End nodes receive half tributary length contributions.

  • Suitable for converting UDLs into lumped nodal loads in structural models.

static TributarySupports(coords, node_ids, load_per_meter)

Compute equivalent nodal reactions (or support tributary forces) for a uniformly distributed load.

This method distributes a uniform line load equally to the nodes of a simple span defined by two end points. It assumes the total load acts over a single segment between the first and last node, and assigns half of the total tributary contribution to each node.

Parameters:

  • coords (array-like) – Nodal coordinates used to compute span length (Euclidean distance between first two nodes).

  • node_ids (list[int]) – Node indices representing the support or boundary nodes of the span.

  • load_per_meter (float) – Uniform distributed load intensity (force per unit length).

Returns:

Mapping of node_id → tributary support force contribution.

Return type:

dict[int, float]

Notes

  • The span length is computed using Euclidean distance between the first two nodes.

  • Each node in node_ids receives an equal share: (total load / 2).

  • This is a simplified distribution model and assumes uniform geometry and loading.
