static Graph(graph, show_weights=False, show_edges_ids=False, show_face_ids=False, show_utilization=False, colored=False, A=0.002848, S=235000000.0, weight_scale=1, font_scale=1, node_scale=1, node_font_scale=1.5, fig_width=8, fig_height=8)

Plot a 2D or 3D representation of a graph.

This function visualizes a graph using NetworkX and Matplotlib. If the graph is 3D, it delegates to a dedicated 3D plotting function.

Parameters:

  • graph (Graph) – Instance of the Graph class to be visualized.

  • show_weights (bool, optional) – If True, display edge weights as labels.

  • show_edges_ids (bool, optional) – If True, display edge IDs.

  • show_face_ids (bool, optional) – If True, highlight faces and display their IDs.

  • show_utilization (bool, optional) – If True, display edge utilization values.

  • colored (bool, optional) – If True, edges are colored and scaled based on force magnitude: - blue for tension - red for compression - black for zero force

  • A (float, optional) – Cross-sectional area used for utilization calculation.

  • S (float, optional) – Yield strength used for utilization calculation.

  • weight_scale (float, optional) – Scaling factor for edge thickness based on force magnitude.

  • font_scale (float, optional) – Scaling factor for edge label font size.

  • node_scale (float, optional) – Scaling factor for node size.

  • node_font_scale (float, optional) – Scaling factor for node label font size.

  • fig_width (float, optional) – Width of the figure in inches.

  • fig_height (float, optional) – Height of the figure in inches.

Returns:

Displays the plot using Matplotlib.

Return type:

None

Notes

  • Requires NetworkX and Matplotlib.

  • For 3D graphs, the function automatically calls PlotGraph3D.

  • Edge utilization is computed via Graph.Utilizations.

  • Face visualization fills polygons with transparency and labels centroids.

static Graph3D(graph, show_weights=True)

Plot a 3D representation of a graph using Plotly.

This function visualizes nodes and edges of a graph in 3D space. Edge labels can display either force magnitudes or edge IDs.

Parameters:

  • graph (Graph) – Instance of the Graph class to be visualized.

  • show_weights (bool, optional) – If True, edge labels display force magnitudes. If False, edge labels display edge IDs. Default is True.

Returns:

Displays an interactive 3D plot using Plotly.

Return type:

None

Notes

  • Requires Plotly (plotly.graph_objects).

  • Nodes are displayed as points, edges as line segments.

  • Node IDs are shown as text labels.

  • Edge labels are positioned at edge midpoints.

  • Includes predefined camera views:

    • Plan (top view)

    • Front view

    • Isometric view

static EdgePerformanceDonut(truss, show_colorbar=True, donut_radius=1, donut_thickness=0.5, font_scale=1, label_offset=0.075, wedge_edgecolor='black', wedge_linewidth=0.1)

Plot a donut chart representing edge performance based on length and axial force.

Each wedge corresponds to an edge in the graph: - Wedge size is proportional to edge length. - Wedge color represents axial force (compression/tension).

Parameters:

  • truss (Truss) – Instance of the Truss class containing edge lengths and forces.

  • show_colorbar (bool, optional) – If True, display a colorbar indicating axial force magnitude.

  • donut_radius (float, optional) – Outer radius of the donut chart.

  • donut_thickness (float, optional) – Thickness of the donut (difference between outer and inner radius).

  • font_scale (float, optional) – Scaling factor for label font size.

  • label_offset (float, optional) – Radial offset for positioning edge labels outside the donut.

  • wedge_edgecolor (str, optional) – Color of wedge borders.

  • wedge_linewidth (float, optional) – Line width of wedge borders.

Returns:

Displays the donut chart using Matplotlib.

Return type:

None

Notes

  • Requires NumPy and Matplotlib.

  • Edge lengths determine angular span of wedges.

  • Axial forces are mapped to colors using a blue-white-red colormap:

    • Blue → compression

    • Red → tension

    • White → near-zero force

  • Edges are sorted by force magnitude for consistent grouping.

static GraphPerformanceSpider(metrics, values_list, scenario_labels=None, colors=None, normalize=True, fill_alpha=0.4, line_width=1.2, label_distance=1.15, label_fontsize=10, grid_levels=6, grid_color='grey', grid_alpha=0.5, grid_linestyle='--', grid_linewidth=0.8, radial_axis_extension=0.05)

Plot a radar (spider) chart for comparing graph performance metrics.

Each polygon represents a scenario (e.g., a truss design), and each axis corresponds to a performance metric.

Parameters:

  • metrics (list of str) – Names of the performance metrics (axes of the radar chart).

  • values_list (list of list of float) – Values for each scenario. Each sublist corresponds to one scenario and must have the same length as metrics.

  • scenario_labels (list of str, optional) – Labels for each scenario. If None, default labels are assigned.

  • colors (list of color-like, optional) – Colors for each scenario polygon. If None, a default colormap is used.

  • normalize (bool, optional) – If True, normalize values per metric across scenarios to [0, 1].

  • fill_alpha (float, optional) – Transparency of the filled polygon areas.

  • line_width (float, optional) – Line width of the scenario outlines.

  • label_distance (float, optional) – Radial distance for positioning metric labels.

  • label_fontsize (int, optional) – Font size of metric labels.

  • grid_levels (int, optional) – Number of concentric grid levels.

  • grid_color (str, optional) – Color of grid lines.

  • grid_alpha (float, optional) – Transparency of grid lines.

  • grid_linestyle (str, optional) – Line style of grid lines.

  • grid_linewidth (float, optional) – Line width of grid lines.

  • radial_axis_extension (float, optional) – Extension beyond the outermost grid for radial axes.

Returns:

Displays the radar chart using Matplotlib.

Return type:

None

Notes

  • Requires NumPy and Matplotlib.

  • Each metric is normalized independently across all scenarios when normalize=True.

  • The chart is drawn in polar coordinates with custom polygonal grids.

  • Suitable for comparing structural performance metrics across multiple designs.

static Graph2Compare(graph1, graph2, highlight_edges1=None, highlight_nodes1=None, highlight_edges2=None, highlight_nodes2=None)

Plot a side-by-side comparison of two graphs.

The function displays four subplots: - Top-left: first graph with edge IDs - Bottom-left: first graph with edge weights - Top-right: second graph with edge IDs - Bottom-right: second graph with edge weights

Parameters:

  • graph1 (Graph) – Original graph instance.

  • graph2 (Graph) – Modified graph instance to compare against the original.

  • highlight_edges1 (list of int, optional) – Indices of edges in graph1 to highlight.

  • highlight_nodes1 (list of int, optional) – Indices of nodes in graph1 to highlight.

  • highlight_edges2 (list of int, optional) – Indices of edges in graph2 to highlight.

  • highlight_nodes2 (list of int, optional) – Indices of nodes in graph2 to highlight.

Returns:

Displays the comparison plot using Matplotlib.

Return type:

None

Notes

  • Requires NetworkX and Matplotlib.

  • Node positions are taken from graph coordinates (assumed 2D).

  • Highlighted nodes and edges are colored differently using a colormap.

  • Edge labels display either IDs (top row) or weights (bottom row).

  • Useful for visualizing structural modifications between two graphs.

static NodeGrid(ground_structure, fig_width=None, fig_height=None, font_scale=1, node_scale=1, node_font_scale=1)

Plot a 2D node grid representation of a ground structure.

This function visualizes the nodal layout of a structured grid using a scatter plot, with node indices annotated for reference. It is primarily used for inspecting connectivity and spatial arrangement of discretized structural systems.

Parameters:

  • ground_structure (object) – Structure containing grid information. Must provide: - nodal_coords : array-like (N x 2) - n_rows : int - n_cols : int - dx : float (grid spacing in x-direction) - dy : float (grid spacing in y-direction)

  • fig_width (float, optional) – Width of the figure. If None, inferred from number of columns.

  • fig_height (float, optional) – Height of the figure. If None, inferred from number of rows.

  • font_scale (float, optional) – Global scaling factor for text elements (title, axis labels).

  • node_scale (float, optional) – Scaling factor for node marker size.

  • node_font_scale (float, optional) – Scaling factor for node index labels.

Returns:

Displays a matplotlib figure and does not return a value.

Return type:

None

Notes

  • Nodes are plotted using a scatter plot with consistent styling.

  • Node indices are annotated slightly offset from their coordinates.

  • Axis ticks are aligned with the grid spacing (dx, dy).

  • Grid styling is intentionally light to emphasize topology.
